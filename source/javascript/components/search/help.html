There are currently 4 ways to search for instances, by ID, name, public IP and DNS address and tag key/value pair, each described in additional detail below.

All searches are case-insensitive, and terms can be put in quotes to correctly search for values with spaces

To search for an instance by ID, you'd simply input the instance ID into the search field. Searching by instance will always return a single instance, and never a list of instances. Wildcards are not supported.

Example

i-91a6404c

To search for instances by name, you can either search by simply typing the name of the instance you want to find, or by using the tag query format. This query can return more than a single instance, if multiple instances have identical names

Examples

inquisitor-asg

or

tag:name=inquisitor-asg

If you need to search for an instance by its public IP address or DNS name, you can simply search for the DNS name or the IP directly

Example

54.186.42.171
or 
ec2-54-186-42-171.us-west-2.compute.amazonaws.com

The most powerful type of search allows the user to search by tag key/value pairs, by searching for tag:key=value.

Examples

Search for instances with the name inquisitor-asg 

tag:name=inquisitor-asg

Search for instances tagged with ElasticSearch CloudTrail cluster as the AppName 

tag:AppName="ElasticSearch CloudTrail cluster"
